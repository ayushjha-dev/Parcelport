# Parcel Registration Fix

## Problem
Parcels were not being saved to the database when students registered them. The form data was lost between steps and never submitted to Firestore.

## Solution Implemented

### 1. Created Parcel Registration Store
**File:** `src/stores/parcelRegistrationStore.ts`

- Uses Zustand for state management
- Persists data across page navigation using localStorage
- Stores all form data from all 4 steps
- Provides methods to update each step's data

### 2. Updated All Registration Steps

#### Step 1 (Personal Information)
- Loads existing data from store
- Saves data to store when continuing
- Pre-fills form with user profile data

#### Step 2 (Parcel Information)
- Loads existing data from store
- Saves tracking ID, courier, description, etc.

#### Step 3 (Time Slot Selection)
- Loads existing data from store
- Saves selected time slot

#### Step 4 (Payment)
- Loads all collected data from previous steps
- Uploads payment screenshot to Firebase Storage
- Submits complete parcel data to `/api/parcels` endpoint
- Creates parcel record in Firestore with status `pending_payment`
- Resets store after successful submission
- Shows loading state during submission
- Displays success/error messages using toast notifications

### 3. Data Flow

```
Step 1 → Store → Step 2 → Store → Step 3 → Store → Step 4 → API → Firestore
```

### 4. Parcel Data Structure

The following data is saved to Firestore:

```typescript
{
  // Student Information
  student_id: string,
  student_name: string,
  student_email: string,
  student_phone: string,
  
  // Delivery Address
  hostel_block: string,
  floor_number: string,
  room_number: string,
  landmark: string,
  
  // Parcel Information
  tracking_id: string,
  courier_company: string,
  description: string,
  weight_range: string,
  expected_date: string,
  is_fragile: boolean,
  
  // Delivery Preferences
  preferred_time_slot: string,
  
  // Payment Information
  payment_method: string,
  transaction_id: string,
  payment_date: string,
  payment_screenshot_url: string,
  payment_amount: 10,
  payment_status: 'pending_verification',
  
  // Auto-generated by API
  drid: string,  // Unique delivery reference ID
  otp: string,   // 6-digit OTP for delivery verification
  status: 'pending_payment',
  created_at: Timestamp,
  updated_at: Timestamp,
}
```

## How It Works Now

1. **Student fills Step 1** → Data saved to Zustand store
2. **Student fills Step 2** → Data saved to Zustand store
3. **Student fills Step 3** → Data saved to Zustand store
4. **Student fills Step 4** → 
   - Payment screenshot uploaded to Firebase Storage
   - All data from Steps 1-4 combined
   - Submitted to `/api/parcels` endpoint
   - Parcel created in Firestore
   - DRID and OTP generated
   - Store reset for next registration

5. **Parcel appears in:**
   - Student's parcels list (`/student/parcels`)
   - Admin's parcels panel (`/admin/parcels`)
   - Admin's payments panel (`/admin/payments`) for verification
